#+title: Introduction

* What is this project?
:PROPERTIES:
:ID:       9b35d7aa-cb8a-433f-b0cb-ba1305740b21
:END:

Breeze is a common lisp development tool implemented in common lisp,
with the goal of being portable to different editors and common lisp
implementation. Currently it is only integrated with Emacs.

This is a personal project that I work on from time to time and use
regularly.

* Features

- *On-the-fly linting* via =flymake=.
  - *Automatic linter fixes* (=breeze-quickfix=)
  - N.B. as of beginning of 2026, there are not many linter rules
    implemented.
- *Snippets*
  - See =M-x breeze-insert-...=
- *Context-aware commands*
  - =breeze-quickinsert= (bound to =M-RET=): suggests applicable
    snippets given the current context.
  - =breeze-quickfix= (bound to =C-.=) fix one linter issues in the
    current top-level form.
  - =breeze-other-file= (=C-c C-o=) and
    =breeze-other-file-other-window= (=C-c o=) to jumb between related
    files (e.g. implementation and test file)
  - N.B. as of beginning of 2026, there are not that many context
    "aware" commands
- *Quick code capture* (=breeze-capture=)
  - creates a new lisp file using a template (configurable) in a
    folder (also configurable), ready for immediate editing.
  - This was inspired by =org-capture=
- *Project creation*
  - =M-x breeze-quickproject=
  - create a new project interactively, using =quickproject=
  - configurable default values
  - the system ="breeze/quickproject"= must be loaded separately.
    - and you must run =M-x breeze-refresh-commands= afterwards
- Implemented in Common Lisp to make porting to other editors possible
  in the future.

Currently, breeze's main interface is Emacs; =breeze.el= adds a
minor-mode (=breeze-minor-mode=) and several commands.
