Wrap implementation-dependent accessor for restarts in CL
#+end_quote

By "restart accessors", they mean functions that take a restart and
returns:
- the restart's report function
- the restart's continuation function
  - the function that is called when the restart is invoked
- the restart's interactive continuation
  - the function that is called when invoking the restart
    interactively
- the restart's test function
  - this function is called to determine if a restart is applicable
    (given a condition)

report, continuation, interactive, test
